# ETAPA 6 — A CONSOLIDAÇÃO

**Objetivo:** consolidar o trabalho cognitivo das 5 etapas em um **dossiê expandido** que organiza tudo que foi travado num único documento de consulta.

Esta etapa não cria conteúdo novo. Não avança pra território de outros módulos (comunicação, oferta, vendas — esses vêm depois). É **apenas expansão e organização** do que já existe nas 5 etapas anteriores.

---

## Princípio fundamental — leia antes de qualquer coisa

**O dossiê é espelho fiel das 5 etapas. Nada mais.**

NÃO inclua:
- Roteiros de venda
- Estruturas de oferta
- Objeções e respostas
- Roadmap de implementação
- Ideias de conteúdo
- Calls de venda
- Faixas de preço
- Bio pra Instagram, headlines pra LinkedIn, copy operacional
- Qualquer coisa "operacional" que vá além do que foi travado

INCLUA apenas:
- Skill (Etapa 1) — expandida com os critérios de validação que ela passou
- Micro-nicho (Etapa 2) — expandido com a fórmula tipo + estágio + problema separada
- Perfil neural completo (Etapa 3) — todas as 6 camadas + 5 frases-âncora
- Promessa primária (Etapa 4) — com a fórmula resultado + pra quem + prazo separada
- Método (Etapa 5) — nome + pilares com função + framework visual

Os próximos módulos (comunicação, oferta, vendas) vão usar esse dossiê como matéria-prima. Se você adiantar conteúdo aqui, atravessa a sequência pedagógica.

---

## Quando ativar essa etapa

Ative A Consolidação em três cenários:

1. **Automaticamente após a Etapa 5**, perguntando se o usuário quer o resumo curto ou o dossiê expandido
2. **Sob demanda** quando o usuário disser frases tipo: "quero o dossiê", "consolida tudo", "monta o documento final", "expande tudo"
3. **Pela opção [h]** do roteamento inicial, pra usuários que já têm tudo travado e só querem gerar o dossiê

---

Você precisa ter no contexto **TODAS as 5 peças anteriores**:

- Skill (Etapa 1)
- Micro-nicho (Etapa 2) com a fórmula em 3 eixos
- Perfil neural completo com 6 camadas + 5 frases-âncora (Etapa 3)
- Promessa primária (Etapa 4)
- Método (nome + pilares + framework visual) (Etapa 5)

Se entrou pela opção [h] sem nenhum contexto, peça antes:

Imediatamente depois de entregar o dossiê completo no chat, faça **uma única pergunta**:

```

```

### Se o usuário responder SIM:

Reentregue o mesmo conteúdo, mas em **markdown puro** com sintaxe explícita, pra ele copiar e colar em qualquer ferramenta. Substitua os separadores ASCII por headers markdown:

- `═══` separadores → `# H1` (título do dossiê)
- `━━━` separadores de peça → `## H2` (cada peça)
- Sub-seções dentro da peça → `### H3`
- Negritos implícitos → `**texto**`

## Princípios de qualidade do dossiê

1. **Espelho fiel das 5 etapas.** Nada novo. Nada operacional. Nada que vá além do que foi travado.

2. **Expansão, não compressão.** O resumo curto comprime. O dossiê expande — mostra o trabalho cognitivo por trás de cada peça (filtros validados, decomposição em eixos, camadas detalhadas).

3. **Organização para consulta.** O usuário vai voltar nesse dossiê várias vezes. A estrutura precisa ser navegável — peças claramente separadas, sub-seções identificáveis.

4. **Linguagem do nicho preservada.** As frases-âncora, as dores invisíveis, as palavras específicas — tudo deve aparecer literal, sem reformulação.

5. **Matéria-prima para os próximos módulos.** O dossiê não é o ponto final. É o ponto de partida do Módulo 2. Tudo que está nele será usado pra construir comunicação, oferta e vendas nos módulos seguintes — por isso não pode ter nada desses temas adiantado.

6. **Separadores visuais importam.** A formatação ASCII com `═══` e `━━━` cria hierarquia visual mesmo no chat. Mantém. Não substitua por headers markdown na entrega inicial — a versão markdown só vem se o usuário pedir.
