# ETAPA 5 — O EMPACOTAMENTO

**Objetivo:** transformar o conjunto (skill + nicho + público + promessa) em um método proprietário com nome, pilares e framework visual.

Esta é a etapa de **integração**. Puxa tudo das 4 etapas anteriores e empacota num ativo único.

---

## Princípios fundamentais

1. **O que não tem nome não existe.** Se o seu processo não tem nome, ele não pode ser referenciado, lembrado, recomendado ou comprado. Nomear cria existência. Nomear cria autoridade.

2. **Concreto vence abstrato.** "Clareza" é abstrato — não cria imagem mental. "A Fatia" é concreto — você vê na cabeça. Concreto gruda. Abstrato evapora.

3. **Três ou quatro pilares é o ponto certo.** Menos que três parece raso. Mais que quatro parece complicado. Três ou quatro é a zona onde o cérebro humano segura o método inteiro na cabeça sem precisar de anotação.

4. **Pilares são verbos ou substantivos concretos.** "Decodificar", "Blindar", "Executar". Nunca "Mindset", "Consciência", "Alinhamento". Os bons pilares são ações ou coisas tangíveis. Quando você lê, entende imediatamente o que vai acontecer.

## Formato de entrega do output

1. **Os 3 ou 4 pilares identificados** — cada um com descrição da função (não só nome)
2. **5 opções de nome pra cada pilar** (curto, concreto, memorável)
3. **5 opções de nome pro método completo** — pode usar fórmulas ("O Método X", "Sistema X", "Protocolo X") ou nomes próprios sem fórmula
4. **Combinação recomendada** (melhor nome de método + melhores nomes de pilares)
5. **Framework visual em ASCII** usando blocos `[]` e setas `→`. Exemplo:

```
┌─────────────────────────────────────────────────────┐
│  PROMESSA: [frase da promessa primária]            │
└─────────────────────────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         ↓               ↓               ↓
    [PILAR 1]       [PILAR 2]       [PILAR 3]
         │               │               │
         ↓               ↓               ↓
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │DOR ATUAL │   │ JORNADA  │   │  ESTADO  │
    │          │ → │          │ → │ DESEJADO │
    │"[frase]" │   │[método]  │   │"[frase]" │
    └──────────┘   └──────────┘   └──────────┘
```

6. **Bio pronta pra Instagram** — frase curta de posicionamento
7. **Pergunta de fechamento obrigatória:**

> "Esse é o produto final do módulo. Quer que eu monte um resumo da sua identidade Money Brand completa pra você colar no workbook?"

---

## Checklist de validação interna

- [ ] São 3 ou 4 pilares, não 2 e não 5
- [ ] Cada pilar é verbo ou substantivo concreto
- [ ] Nenhum pilar usa palavras como "Mindset", "Clareza", "Estratégia", "Consciência", "Alinhamento"
- [ ] Nome do método é curto, concreto, memorável
- [ ] Nome do método não soa como curso barato ("Sucesso Total", "Resultado Garantido")
- [ ] Framework visual cabe num desenho simples e legível
- [ ] A bio pronta está em português brasileiro natural (a menos que o nicho justifique inglês)
